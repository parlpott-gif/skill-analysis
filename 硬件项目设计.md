# 硬件项目设计 - AI 驱动工程自动化

## 核心理念

通过 AI（Claude）将硬件项目设计流程**工程化和自动化**，从需求分析到文档生成的全流程标准化。

---

## 工作流程架构

### 第一步：需求分析与元器件清单

**AI 自动执行：**
- 从用户需求中提炼系统名称
- 生成标准化元器件清单
- 应用命名规范（便于后续自动化处理）
- 定义完整的硬件功能需求

**核心规范化处理：**
- 元器件格式标准化
- 执行器与驱动器分离定义
- 主控器件标准化选择

### 第一步b：智能项目相似度检查

**AI 自动执行：**
- 扫描历史项目库
- 基于传感器型号进行相似度匹配
- 若发现相似项目，生成差异对比表
- 提供硬件复用建议和兼容性判断

**价值：** 加速设计决策，提高项目复用率

### 第二步：功能讨论与需求确认

**AI 交互式处理：**
- 根据元器件自动列举可实现功能
- 与用户确认功能需求
- 明确人机交互设计（按键、显示等）
- 对于联网项目，询问上位机形式（APP/Windows）并规划架构

**输出：** 功能清单 + 总表 C 列预览

### 第三步：IO 口智能分配

**AI 根据主控型号自动执行：**
- 选择对应的引脚分配规则
- 优化传感器与执行器分布
- 避免引脚冲突和信号干扰
- 输出标准化的 IO 分配表

**支持主控：** STM32F103C8T6、ESP32-S3

### 第四步：自动生成技术文档

**AI 生成：**
- IO 口分配文档（DOCX 格式）
- PRD 完整文档
  - 硬件资源清单
  - 驱动层能力定义
  - 显示层交互设计
  - 通信协议（如需联网）
  - 功能实现逻辑
  - 上位机开发规范

**特点：** 所有文档结构和字段已标准化，便于后续代码生成

### 第五步：自动化项目管理

**AI 执行：**
- 创建项目文件夹结构
- 更新项目总表（系统名、元器件、功能、链接）
- 自动查询器件价格库，计算硬件成本
- 输出成本报告

**价值：** 中央化项目管理，便于查询和复用

### 第六步：质量检查

**AI 自动验证：**
- 所有文档已生成
- 必填字段已完整
- 文件链接已正确建立
- 提示任何缺失或问题

## Skill 协作模式

该 Skill 与其他工具的配合：

- **与 stm32-hal Skill 配合**：PRD 中的硬件规范直接对应 STM32 驱动库的接口定义
- **与 esp32-arduino Skill 配合**：ESP32 项目的 IO 分配和通信协议无缝适配 ESP32 驱动框架
- **与项目管理工具配合**：自动生成的总表可直接用于项目进度跟踪和成本管理
- **与代码生成工具配合**：标准化的 PRD 文档为后续代码生成提供完整的输入参数

## 工程价值

**设计效率：**
- 传统流程：需求 → 方案讨论 → 文档 → 反复修改（2-3 周）
- AI 流程：需求 → AI 自动生成标准方案 + 文档（1-2 小时）

**质量保证：**
- 规范化的命名和结构
- 自动化的检查机制
- 历史项目的数据支撑

**知识积累：**
- 每个项目都为库增加新数据
- 相似项目越多，复用机会越大
- 形成闭环的工程方法论

---

## 技术栈

- **AI 驱动**：Claude API 负责需求解析、方案生成、文档编写
- **文档自动化**：Python 脚本生成 DOCX、XLS、Markdown
- **库存管理**：JSON 格式的器件库和价格库
- **项目管理**：Excel 中央总表，支持超链接追踪
