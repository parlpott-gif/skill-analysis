# AI-Driven Hardware Engineering System

一个通过 **AI（Claude）驱动的硬件项目设计和开发完整解决方案**，展示如何将工程流程自动化、标准化和工具化。

## 🎯 核心价值

传统硬件项目开发流程：需求 → 器件选择 → IO分配 → 文档编写 → 驱动开发 → 网络集成 ⏱️ **2-3周**

**本系统**：需求 → AI自动生成全套方案 + 文档 + 代码框架 ⏱️ **1-2小时**

---

## 📋 系统组成

### 1. **硬件项目设计 Skill**
`硬件项目设计.md`

**功能**：从硬件需求到完整工程文档的一键生成

工作流程：
```
用户需求
  ↓
AI需求分析 → 元器件清单生成
  ↓
智能相似项目检查 → 复用建议
  ↓
功能讨论 → 需求确认
  ↓
IO口智能分配 → 引脚优化
  ↓
自动文档生成 → PRD + IO分配表
  ↓
项目管理 → 总表更新 + 成本计算
```

**输出文件**：
- IO 口分配文档（DOCX）
- PRD 完整需求文档（Markdown）
- 项目总表（Excel）
- 硬件成本报告

**核心能力**：
- ✅ 自动识别相似项目（基于传感器型号）
- ✅ 智能 IO 口分配（支持 STM32/ESP32）
- ✅ 标准化命名规范
- ✅ 通信协议自动定义
- ✅ 成本自动计算

---

### 2. **STM32 HAL 驱动开发 Skill**
`stm32-hal/SKILL.md`

**功能**：为 STM32 项目自动生成器件驱动代码

**支持的通信总线**：
- GPIO 数字 I/O
- I2C 多器件总线
- SPI 高速通信
- UART 串口通信
- 单总线（OneWire）
- ADC 模拟采集

**代码规范**：
```
项目/
├── Core/
│   ├── Inc/main.h
│   └── Src/main.c
├── APP/
│   ├── app_dht11.c
│   ├── app_dht11.h
│   ├── app_ssd1306.c
│   ├── app_ssd1306.h
│   └── ...
└── stm32-xxx.ioc
```

**工作流程**：
1. 用户提供器件型号 + 总线类型
2. AI 检查资料库中的参考实现
3. 自动生成符合规范的驱动文件
4. 输出 main.c 集成示例

---

### 3. **ESP32 Arduino 驱动开发 Skill**
`esp32-arduino/SKILL.md`

**功能**：为 ESP32 Arduino 项目自动生成器件驱动代码

**特点**：
- Arduino IDE 完全兼容
- 非阻塞 I/O（不影响 WiFi 通信）
- 多器件同时驱动
- 完整错误处理

**支持的通信总线**：同 STM32（GPIO、I2C、SPI、UART、OneWire、ADC、PWM）

---

### 4. **ESP32 MQTT 一键集成 Skill**
`esp32-mqtt.md`

**功能**：为 ESP32 项目一键集成 MQTT 网络功能

**工作流程**：
```
原始代码 + 需求描述
  ↓
AI 自动生成 MQTT 库
  ↓
修改后的代码 + 完整示例
  ↓
开发者部署使用
```

**核心能力**：
- ✅ WiFi 自动连接
- ✅ MQTT Broker 连接管理
- ✅ 自动断线重连
- ✅ 多 Topic 支持
- ✅ 数据上传和指令下发

---

### 5. **OneNET MQTT 对接 Skill**
`OneNET-MQTT.md`

**功能**：为硬件项目一键对接中国移动 OneNET 物联网平台

**支持的接入方式**：
- ESP32 + WiFi MQTT
- STM32 + GSM/NB-IoT
- 其他主控 + 通信模块

**自动对接内容**：
- OneNET 协议适配
- 数据格式标准化
- 指令解析逻辑
- 时间同步

---

### 6. **驱动库扫描反哺 Skill**
`扫描反哺驱动库/SKILL.md`

**功能**：自动扫描项目中的驱动文件，更新库存

**工作流程**：
1. 扫描当前项目的驱动代码
2. 自动识别主控类型（STM32/ESP32）
3. 识别缺失的器件驱动
4. 反哺到对应 Skill 的库存中

## 🔄 Skill 协作架构

```
┌─────────────────────────────────────────┐
│     用户硬件需求描述                    │
└────────────────┬────────────────────────┘
                 │
                 ↓
        ┌────────────────────┐
        │ 硬件项目设计 Skill │
        │ - 元器件清单       │
        │ - IO 口分配        │
        │ - PRD 生成         │
        └────────┬───────────┘
                 │
    ┌────────────┼────────────┐
    │            │            │
    ↓            ↓            ↓
  STM32     ESP32 Arduino   其他主控
  HAL       驱动开发        驱动库
  Skill     Skill          (扩展)
    │            │            │
    └────────────┼────────────┘
                 │
                 ↓
         (可选) MQTT 集成
                 │
    ┌────────────┼────────────┐
    │            │            │
    ↓            ↓            ↓
  ESP32MQTT  OneNETMQTT   其他云平台
  Skill      Skill        (扩展)
    │            │            │
    └────────────┼────────────┘
                 │
                 ↓
          最终完整项目
```

**工作流程特点**：
- 🔹 **上游流**：硬件设计 Skill 定义规范 → 下游 Skills 严格遵守
- 🔹 **中游流**：驱动开发 Skills 生成标准代码
- 🔹 **下游流**：网络 Skills 自动集成通信功能
- 🔹 **反馈流**：扫描反哺 Skill 更新库存，增强后续复用率

## 🎓 学习和扩展

### 如何理解这个系统

1. **从硬件项目设计开始**
   - 了解硬件工程的标准流程
   - 理解为什么需要规范化和自动化

2. **了解各 Skill 的职责**
   - STM32/ESP32 Skill：代码生成
   - MQTT Skill：网络集成
   - 扫描反哺：知识库维护

3. **看工作流程的协作**
   - 上游定义 → 下游执行
   - 数据流向和反馈机制

### 如何扩展

- **新主控支持**：参考 STM32 HAL Skill 的规范，创建新 Skill
- **新通信协议**：添加到各驱动 Skill 的总线列表
- **新云平台**：参考 OneNET MQTT Skill 的模式
- **新器件驱动**：向库中补充参考实现
