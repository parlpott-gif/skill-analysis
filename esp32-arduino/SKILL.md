# ESP32 Arduino 驱动开发助手

## 核心功能

为 ESP32 Arduino 项目**快速生成标准化的驱动代码**。

### 生成内容

| 内容 | 说明 |
|------|------|
| 驱动文件 (.cpp/.h) | 单个器件的完整驱动实现 |
| 接口定义 | 标准化的初始化、读写、控制接口 |
| 配置示例 | .ino 主程序中的调用示例 |
| 文档 | 驱动使用说明和 GPIO 映射表 |

---

## 工作流程

### 第一步：器件和总线确认

用户提供：
- 器件型号（如 DHT11、SSD1306、MAX30102）
- 通信总线（GPIO、I2C、SPI、UART、单总线）
- GPIO 分配（基于硬件设计 Skill 的 IO 分配文档）

### 第二步：AI 自动生成

AI 检查资料库，选择：
- 已有的参考实现（直接适配）
- 总线模板骨架（从模板扩展）
- 新器件规范（从零开发）

输出完整的 Arduino 兼容驱动代码。

### 第三步：开发者集成

- 复制生成的 .cpp/.h 到项目的 lib/ 或 src/ 目录
- 在 .ino 主程序中调用初始化和功能函数
- 在 Arduino IDE 中编译上传

## 技术规范

**代码规范：**
- Arduino IDE 兼容（.cpp/.h 和 .ino 格式）
- 类化接口，易于使用和扩展
- 支持多个同类器件（如多个 DHT11 传感器）
- 完整的错误处理

**通信协议支持：**
- GPIO 数字 I/O
- I2C 多器件总线
- SPI 高速通信
- UART 串口通信（包括 Serial 调试）
- 单总线（Dallas OneWire）
- ADC 模拟采集
- PWM 输出

**内存优化：**
- 最小化内存占用（ESP32 有 520KB SRAM）
- 高效的中断处理
- 可配置的缓冲和采样周期

**WiFi/网络友好：**
- 非阻塞 I/O（不影响 WiFi 通信）
- 支持 MQTT 集成（与 ESP32 MQTT Skill 配合）
- 低功耗设计选项

## 与硬件项目设计 Skill 的配合

**集成流程：**

1. **硬件设计**：定义所有器件及其总线（来自硬件项目设计 Skill）
2. **IO 分配**：确定各器件的 GPIO（来自硬件项目设计 Skill）
3. **驱动生成**：ESP32 Arduino Skill 根据器件列表逐一生成驱动
4. **代码集成**：开发者整合所有驱动到主程序
5. **功能实现**：根据 PRD 实现业务逻辑
6. **网络集成**：（可选）使用 ESP32 MQTT Skill 增加联网功能

**优势：**
- 硬件规范 → 驱动代码无缝转换
- GPIO 映射自动验证（避免冲突）
- 开发时间大幅减少
- 代码风格统一，易于维护
