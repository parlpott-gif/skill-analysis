# 硬件项目设计流程规范

## 元器件命名规则

### B列元器件统一规范

**通用外设统一命名：**
- 继电器统一写 `继电器`，不加"模块/单路"等后缀
- 步进电机固定写 `28BYJ-48步进电机` 和 `ULN2003` 合并为一行：`28BYJ-48+ULN2003`

**B列只填真实元器件，禁止填通信接口/协议描述：**
- ❌ 错误：`UART1上报`、`I2C总线`、`SPI接口`、`PA9通信`
- ✅ 正确：写模块型号，如 `BC26`（NB-IoT）、`ESP8266`（WiFi）、`SIM800C`（GSM）
- 联网模块写模块芯片型号，通信方式（UART/SPI等）只体现在括号里的引脚，不单独成行

**按键规则：**
- 一使用就是4个，就四个，不多也不少

2. 对比每行 B列元器件 与本次元器件清单，按**传感器器件型号重合度**判断相似性
3. 若存在相似项目（传感器重合2个及以上），输出差异对比表，格式如下：

**发现相似项目：《{已有系统名}》**

| 对比项 | 总表已有 | 本次设计 |
|--------|----------|----------|
| 传感器 | ... | ... |
| 执行器/继电器 | ... | ... |
| 通信模块 | ... | ... |
| 按键数量 | ... | ... |
| 其他 | ... | ... |

**硬件差异说明：**
- 列出哪些差异属于硬件层面（器件不同、引脚数不同、模块规格不同）
- 哪些属于软件层面（功能逻辑、阈值、控制方式不同）
- 硬件差异需要采购不同器件，软件差异可复用代码修改解决

**引脚兼容性判断：**
对每处器件差异，额外标注是否引脚兼容：
- ✅ 引脚兼容：引脚数相同、排列相同（如 DHT11 vs DS18B20 均为3pin VCC/DATA/GND，MQ系列模块均为4pin AO/DO/VCC/GND），板子不用改，只换器件+改驱动代码
- ⚠️ 引脚不兼容：引脚数或通信方式不同（如单总线 vs I2C vs UART），需要重新飞线或更换接口

最终给出一句总结：**"板子可直接复用 / 需要部分改线 / 需要重新布线"**

4. 若无相似项目，不输出任何内容，直接进入第二步
5. 输出差异后，**继续等待用户确认元器件**，不阻塞流程

### 第三步：IO口分配（等待确认）

- 用户确认元器件后，进行IO口分配
- **根据主控型号选择对应规则**

#### STM32F103C8T6 项目

- ADC：PA0-PA7（ADC1）
- I2C1：PB6(SCL) / PB7(SDA)
- 避免IO口冲突
- 分类展示：ADC模拟输入 / 数字输入 / 数字输出 / I2C

**IO口分配原则（引脚侧分布）：**
- 传感器（ADC输入、数字输入）优先分配到 PA 侧引脚（PA0-PA7 ADC，PA8-PA15 数字）
- 驱动器（数字输出：继电器、LED、蜂鸣器、舵机、电机等）优先分配到 PB 侧引脚
- 按键固定使用 PA15 / PB3 / PB4 / PB5，不使用其他引脚
- I2C 固定 PB6(SCL)/PB7(SDA)，OLED/LCD 等显示模块归入 PB 侧
- **步进电机（28BYJ-48+ULN2003）固定使用 PB13 / PB14 / PB15 / PA8**，不使用其他引脚

#### ESP32-S3-N8R8 项目

- **可用GPIO范围：GPIO1-18、GPIO21、GPIO38-48**
- **禁用引脚：GPIO19/GPIO20（USB D-/D+，不可用作普通GPIO）**
- ADC1（WiFi启用后ADC2不可用）：GPIO1-10
- I2C 默认：GPIO8(SDA) / GPIO9(SCL)

**IO口分配原则：**
- 模拟传感器（AO）→ GPIO1-7（ADC1范围，避开I2C的8/9）
- 数字传感器（单总线/DO）→ GPIO4、GPIO5、GPIO6（ADC1范围，可复用为数字输入）
- 按键固定使用 GPIO11 / GPIO12 / GPIO13 / GPIO14
- 数字输出（继电器、LED、蜂鸣器、电机等）→ GPIO15-18，不够用时顺延到 GPIO21、GPIO38+
- I2C 固定 GPIO8(SDA)/GPIO9(SCL)
- 步进电机4线优先用连续引脚，跳过19/20，可用 GPIO15/16/17/18 或 GPIO17/18/21/39（蜂鸣器用GPIO38）
