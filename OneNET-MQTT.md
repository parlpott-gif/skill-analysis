# OneNET MQTT 一键对接 Skill

## 功能概述

为 STM32 或 ESP32 项目**一键对接中国移动 OneNET 物联网平台**。

**核心价值：** 将复杂的 OneNET 对接流程自动化，开发者只需提供业务需求，AI 生成完整的网络通信代码。

---

## 支持的对接方式

| 平台 | 主控 | 通信方式 | 特点 |
|------|------|--------|------|
| OneNET | STM32 | MQTT over TCP | 稳定可靠 |
| OneNET | ESP32 | MQTT over WiFi | 无线便捷 |
| OneNET | NB-IoT 模块 | MQTT over LTE-M | 低功耗物联网 |

---

## 工作流程

### 第一步：需求定义

用户提供：
- 设备类型（温度传感器、环境监测站等）
- 上报的数据字段（温度、湿度、PM2.5 等）
- 接收的控制指令（阈值设置、启停控制等）
- OneNET 项目 ID 和接入凭证

### 第二步：AI 自动生成

AI 输出：
- OneNET 协议适配层（处理与 OneNET 通信的细节）
- 数据上报格式定义
- 指令解析逻辑
- 修改后的主程序

### 第三步：开发者配置和部署

- 填入 OneNET 项目 ID、API Key
- 修改数据上报周期和阈值
- 编译上传到硬件
- 在 OneNET 平台端验证数据

---

## 核心能力

| 能力 | 说明 |
|------|------|
| 平台对接 | 自动适配 OneNET MQTT 的协议标准 |
| 数据格式 | JSON/二进制灵活编码 |
| 指令处理 | 自动解析和分发云端指令 |
| 断线处理 | 重连机制和离线缓存 |
| 时间同步 | NTP 时间获取（确保时间戳准确） |

## 与硬件项目设计 Skill 的配合

**流程关联：**

1. **硬件设计阶段**（硬件项目设计 Skill）
   - 定义传感器和执行器
   - 规划通信协议（上报数据字段、下发指令）
   - 输出 PRD 文档包含通信规范

2. **驱动开发阶段**（STM32/ESP32 驱动库）
   - 编写单个传感器/执行器的驱动
   - 实现业务逻辑（数据采集、阈值判断）

3. **网络集成阶段**（OneNET MQTT Skill）
   - 自动生成通信代码
   - 与已定义的协议无缝衔接
   - 快速上线物联网功能

---

## 技术特点

**设计理念：**
- 协议细节对开发者透明
- 提供标准库接口
- 支持离线工作和数据同步
- 低功耗设计（适配电池供电场景）

**部署灵活性：**
- 支持多个 OneNET 项目同时对接
- 支持 MQTT 和其他协议混合使用
- 易于扩展新的数据类型和指令
