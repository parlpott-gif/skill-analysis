# STM32 HAL 库驱动开发助手

## 核心功能

为 STM32 硬件项目**快速生成标准化的驱动代码**。

### 生成内容

| 内容 | 说明 |
|------|------|
| 驱动文件 (.c/.h) | 单个器件的完整驱动实现 |
| 接口定义 | 标准化的初始化、读写、控制接口 |
| 配置示例 | main.c 中的调用示例 |
| 文档 | 驱动使用说明和接口文档 |

---

## 工作流程

### 第一步：器件和总线确认

用户提供：
- 器件型号（如 DHT11、DS1302、INA226）
- 通信总线（GPIO、I2C、SPI、UART、单总线）
- 项目架构（STM32F103C8T6 + CubeMX 工程）

### 第二步：AI 自动生成

AI 检查资料库，选择：
- 已有的参考实现（直接适配）
- 总线模板骨架（从模板扩展）
- 新器件规范（从零开发）

输出完整的驱动代码。

### 第三步：开发者集成

- 复制生成的 .c/.h 到 APP/ 文件夹
- 在 main.c 中调用初始化和功能函数
- 编译验证

## 技术规范

**代码规范：**
- 每个器件一对 app_器件名.c/.h 文件
- 接口函数以 APP_ 前缀统一调用
- 无硬件初始化细节暴露给 main.c

**通信协议支持：**
- GPIO 数字 I/O
- I2C 多器件总线
- SPI 高速通信
- UART 串口通信
- 单总线（Dallas）
- ADC 模拟采集

**内存优化：**
- 最小化全局变量
- 支持中断驱动
- 可配置缓冲区大小

## 与硬件项目设计 Skill 的配合

**集成流程：**

1. **硬件设计**：定义所有器件及其总线（来自硬件项目设计 Skill）
2. **IO 分配**：确定各器件的引脚（来自硬件项目设计 Skill）
3. **驱动生成**：STM32 HAL Skill 根据器件列表逐一生成驱动
4. **代码集成**：开发者整合所有驱动到主程序
5. **功能实现**：根据 PRD 实现业务逻辑

**优势：**
- 硬件规范 → 驱动代码无缝转换
- 开发时间从天级降至小时级
